Behind the scenes — Systematic evidence search
Three-strategy PubMed search — guidelines, technical parameters, protocol optimization
PubMed is the US National Library of Medicine's database of 36M+ biomedical papers. Three complementary searches (clinical guidelines via MeSH medical headings, technical parameter papers, protocol optimization studies) ensure we don't miss relevant evidence — the same approach used in formal systematic reviews.
↓
Up to 60 papers pooled and deduplicated
Each of the three search strategies returns up to 20 results. Duplicates across the three searches are removed, leaving a single evidence pool filtered to 2018-onward literature.
↓
Abstracts fetched for entire pool
↓
PMC full-text Methods sections fetched where Open Access allows
PubMed abstracts rarely contain parameter tables — those live in the Methods section of full papers. For papers with a PMC Open Access ID, the system fetches and appends the Methods text, enriching both the density scorer and the LLM context.
↓
Parameter-density scorer — counts TE, TR, TI, ms, mm, bandwidth, NEX terms per abstract
Most PubMed abstracts about MS or Stroke are clinical outcome papers that don't contain actual MRI numbers. This scorer uses pattern matching to count explicit technical terms (e.g. "TE = 80 ms", "slice thickness 3 mm") in each abstract, so papers with real parameter data are prioritised over those that only discuss clinical findings.
↓
Composite ranking: Gold journal > Recency > Param density > Date
Gold journals are the top 7 radiology publications (Radiology, AJNR, JMRI, European Radiology, RadioGraphics, Investigative Radiology, AJR) — their data is given the highest trust. Within each tier, newer papers with more technical content rank higher.
↓
Top 5 abstracts + optional OEM vendor guide + optional site reference sent to LLM with GRADE rubric
GRADE is the international standard for rating evidence certainty, used by WHO, Cochrane, and clinical guideline bodies worldwide. The LLM is instructed to rate its own output as High, Moderate, or Low based on how many of the provided abstracts contain explicit MRI parameter numbers from gold-standard journals. OEM vendor protocol guides provide manufacturer-recommended defaults (ranked below PubMed but above model training knowledge). A parameter-rich user reference document can independently justify High or Moderate evidence strength.
↓
LLM generates rules.json — TE/TR ranges, target durations, clinical rationale, evidence strength
The output is a structured benchmark file containing: acceptable TE/TR ranges per sequence (e.g. FLAIR TE: 80–140 ms), target acquisition durations, a clinical rationale summary, and a GRADE-aligned evidence strength rating. This file is what every uploaded DICOM is compared against.

DICOM parser extracts physics-only tags: TE, TR, Flip Angle, TI, Duration, Matrix, Slice Thickness, Pixel Spacing, Field Strength
DICOM is the standard file format for medical images. Each file contains hundreds of metadata tags. The parser reads only the technical sequence tags — timing (TE, TR, TI), contrast (flip angle), spatial resolution (slice thickness, pixel spacing, voxel size, FOV), and hardware context. It deliberately ignores patient name, DOB, and all other Protected Health Information (PHI).
↓
Temp file deleted immediately
The uploaded file is removed from disk as soon as the physics tags are extracted. No patient data is retained.
↓
Step 3 — Review the audit
Series matching — exact match or AI semantic mapping — with confidence rating
Scanner vendors use inconsistent naming (e.g. "T1 3D 0" vs "3D T1 MPRAGE SAG"). If the series name doesn't exactly match a rule, the system asks an AI model to find the closest clinical equivalent — like a translator between vendor dialects and standard terminology. Each match is assigned a confidence level (high for exact, medium for semantic) and a benchmark rationale explaining why this comparison was chosen.
↓
Yes →
PASS — physics OK
← No
FAIL — protocol drift detected with expected vs actual values
Protocol drift is the gradual divergence of scanner parameters from agreed standards, usually caused by manual tweaks over weeks or months. Even small TE/TR changes can silently degrade image contrast.
↓
Efficiency score — compare AcquisitionDuration vs target_duration_ms from PubMed benchmark
The scan's actual acquisition time (extracted from the DICOM file) is compared to the target duration derived from current PubMed evidence. A score of 100 means the scan matches or beats the benchmark.
↓
Yes →
Revenue opportunity alert displayed
← No
Efficiency within acceptable range
A scan running significantly slower than modern benchmarks represents lost scanner capacity. Modernising sequence physics could add on the order of one additional billable patient slot per day when scans are materially slower than benchmark.
↓
Clinical grade assigned: A / B / C / F based on physics + efficiency combined
A = physics pass + efficient. B = physics pass + no efficiency data or minor delay. C = physics pass + significantly slow. F = physics fail (TE/TR outside acceptable range).
↓
1.5T hardware warning if aggressive 3T params detected on 1.5T scanner
Running 3T-optimised parameters on a 1.5T scanner reduces signal-to-noise ratio, potentially producing grainy images. The system detects when TE/TR sit in the aggressive low end of the acceptable range on lower-field hardware.
↓
What the radiologist sees
Clinical rationale panel — executive summary, GRADE-aligned evidence badge, recent literature changes
This panel shows the AI's reasoning: what the evidence says, how strong it is (High/Moderate/Low), and what has changed in recent literature versus older protocols — so the radiologist can judge whether to trust the recommendation.
↓
PubMed sources table — year, journal, gold star, param density, PMID link
Full provenance for every paper used: publication year, journal name, whether it's a gold-standard source, how many technical terms its abstract contains, and a direct link to the original paper on PubMed. If a user-provided reference document was included, it appears as a separate row with its own parameter density score.
↓
Low-evidence tip (when strength is Low) — refine query, attach reference PDF, verify locally
When PubMed abstracts lack explicit parameters and no reference document was attached, an actionable tip guides the radiologist on how to improve evidence quality for the next sync.
↓
Sequence parameters — hardware, series, TE, TR, acquisition duration
↓
Sequence identity row — flip angle, inversion time (TI), benchmark protocol label
These three fields help the radiologist verify whether the sequence type is correctly identified. Flip angle and TI are essential for distinguishing IR-prepared volumes (e.g. MPRAGE) from spoiled gradient-echo sequences that may share similar TE/TR values.
↓
Acquisition context — matrix, averages, slice thickness, spacing, voxel size, FOV, pixel spacing
Spatial resolution details let the radiologist judge whether a protocol compromise happened in the spatial domain (thicker slices, larger voxels) rather than just the timing domain. FOV is derived from matrix × pixel spacing when the DICOM tags are present.
↓
Series mapping with confidence — match method, confidence indicator, benchmark rationale
Displays how the scanner's series label was mapped to a benchmark protocol: exact match (high confidence) or AI semantic mapping (medium confidence), along with a plain-language rationale. This transparency helps radiologists judge whether a "drift" finding reflects a real problem or a mapping artefact.
